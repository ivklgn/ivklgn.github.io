Hotpot Tracker – Experimental Project and Goal Management System

Russian version available here

Hotpot Tracker is an experimental system designed to explore improvements to project/goal management tools such as Jira, Trello, Asana, and others. The project concept combines familiar features with what I believe to be breakthrough ideas for optimizing task management, visualization, and control.

Main objectives of the experiments:

  1. Navigation and search: How can we quickly find relevant tasks while retaining the "task board" paradigm?
  2. Increasing team engagement: What do existing systems lack to prevent users from migrating to chats?
  3. What defines a "Task"? A new perspective on this familiar term opens new possibilities.
  4. Optimizing communication: How can task-related communication be made easier and clearer?

Concept

Disclaimer:
This system intentionally avoids complex features like timelines or performance reports, focusing instead on other productivity-boosting aspects.
  1. Multiple boards instead of one
  2. Attributes for boards and tasks instead of folders
  3. Everyone can participate
  4. Approvals instead of task status transition rules
  5. Linked attributes
  6. Remarks instead of comments in development

Multiple Boards Instead of One

Instead of managing everything on a single overloaded board, you can create separate boards for different goals and tasks. In my experience, many companies using Kanban/Scrum often rely on a single team board with goals represented as epics or milestones.

Managing parallel processes on a single board increases task management complexity and cognitive load for the team.

Idea:
Each employee or team creates their own boards tailored to their needs, simplifying navigation and management. For example, one board could be for ongoing tasks, another for long-term objectives (OKRs), and another for incident management.

Creating boards should be quick and easy—any team member can set up a board as needed. Multiple boards shouldn’t be intimidating, as each can be opened independently to maintain focus during discussions and work.

Systems like Sentry or Graylog offer inspiration on how search and filtering can be effective. In Hotpot Tracker, filtering boards is simple and efficient using attributes, though task-level search is not yet implemented.

Attributes for Boards and Tasks

Hotpot Tracker is built around three core entities:

  1. Task – title, content, attributes, and a reference to a column. In Hotpot Tracker, a task is primarily treated as a document.
  2. Column (Stage) – status, participants, and a reference to a board.
  3. Board – name.
Idea:
A unified attribute system for boards and tasks creates a consistent interface for editing meta-information. Hotpot does not impose predefined attributes on users.

This approach solves two problems:

  1. Finding tasks is easier using attribute filters.
  2. Attributes simplify management by making it clear who is responsible for tasks and their deadlines.
Idea:
Board attributes are automatically inherited by tasks. For example, if a board has a "Team" attribute, the tasks on that board will display the same attribute.

Everyone Can Participate

By 2024, it became clear that product management and development are moving in the same direction but operate differently. Product representatives often don't participate directly in development, complicating requirement management. A valuable improvement is functionality that allows indirect participation through boards.

Idea:
Stage = Status + Participants. Each column can be linked to specific participants, such as assigning reviewers or approvers.

This feature rethinks the collaboration approach, inviting product or analytics representatives to contribute within technical teams.

Approvals Instead of Status Transition Rules

Idea:
Instead of complex status transition rules (like in Jira), Hotpot Tracker uses a simple approval system.

Approvals make the process flexible and transparent, similar to GitLab’s review system, where changes are approved before completion.

Linked Attributes

Idea:
Task attributes (e.g., deadlines, priorities) can be linked to specific contributors.

Remarks Instead of Comments in development

Traditional comments often create noise. Hotpot Tracker replaces them with remarks tied to specific task parts, similar to inline comments in version control systems like GitLab.

Conclusion

Based on the goals of the experiments, I can conclude the following:

  1. Navigation and search: How can we quickly find the necessary tasks while staying within the "task board" paradigm?

    Attributes for boards/tasks may provide a more convenient UX than full-text search or arranging boards/tasks into different spaces.

  2. Increasing employee engagement: What do current systems lack that causes users to resort to chats?

    The ability to assign participants (contributors) to columns allows involving the right people for collaborative decision-making.

  3. What is a "Task"?: A different perspective on such a familiar term opens up new possibilities.

    A task is a document + attributes + approval. The focus should be on working on the task's content (its completeness and accuracy), rather than treating the task as a mere formality.

  4. Optimal communication: How can we make task-related communication simpler and clearer?

    Moving from regular comments to discussions tied to the task's text will increase the value of communication.
